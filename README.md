## Roles and first super admin

The API has three roles: `USER` (can book trainings), `SCHOOL_ADMIN` (can manage owned schools, their trainings, gallery and news) and `SUPER_ADMIN` (can manage every school).

Accounts are created through `POST /api/auth/register`; the API uses HTTP Basic authentication for protected operations. To create or promote the first super administrator on application startup, provide both environment variables:

```bash
BARKLY_ADMIN_EMAIL=you@example.com BARKLY_ADMIN_PASSWORD='a-long-unique-password' ./gradlew -p api bootRun
```

School slugs are generated by the server from the name and the database ID, for example `Psia Akademia` becomes `psia-akademia-42`. They are never entered by a school administrator.
